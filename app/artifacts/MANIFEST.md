# app/artifacts/ — MANIFEST

Corpus artifacts that the app loads at startup. Two layers: prompt
text (`voice_card.md`, `router_prompt.md`) consumed as system prompts
by the Claude calls, and synthesis JSON (topic map, graph, entity index,
frameworks, signature library) consumed by `build_context()` and the
router. The per-doc Stage-1 extractions live under `topics/`.

| ID | File | Description |
|---|---|---|
| 0001 | [voice_card.md](voice_card.md) | Inference system prompt — identity, voice fingerprint, register, out-of-corpus policy, worked example. 229 lines, ~3,265 tokens. Cached via `cache_control` on the Sonnet call. Edits change CJ's voice; treat the build-kit copy at `corpus/build_kit/voice_card.md` as upstream. |
| 0002 | [router_prompt.md](router_prompt.md) | Router system prompt — 198 lines, ~2,400 tokens. First triple-backtick block is parsed as the system block. Edits change topic-routing behavior. |
| 0003 | [topic_map.json](topic_map.json) | 37 canonical topics organized in anchor / pillar / vector / personal tiers, each with display name, definition, doc_ids, stances, signature phrases. Loaded by `build_context()` to assemble the inference context. |
| 0004 | [topic_graph.json](topic_graph.json) | 37 nodes, 78 weighted edges — which topics co-occur in the corpus. Loaded but not currently consumed by inference. |
| 0005 | [entity_index.json](entity_index.json) | 69 people, 16 cases, plus laws and treaties — canonicalized aliases. Loaded but used selectively. |
| 0006 | [frameworks.json](frameworks.json) | 10 named mnemonic frameworks (Four Ins, Three Es, twin beacons, chiastic doublets, etc.). |
| 0007 | [signature_library.json](signature_library.json) | 684 normalized signature phrases. Loaded by `CorpusArtifacts` but **not currently referenced by `build_context()`** — see [LL-005](../../docs/lessons/LL-005-signature-library-loaded-but-unused.md). |

## Subdirectories

| ID | Path | Description |
|---|---|---|
| S0001 | [topics/](topics/) | 89 Stage-1 per-doc topic extractions, one JSON per source document — `book_01_front-matter.json`, `book_01_ch01.json` … `book_01_appendix-d.json` (25 files from the book), plus `col_YYYY_MMDD.json` (65 files from the columns). Each holds title, date, voice register, primary topics, stances, signature phrases, notable anecdotes, citations. `build_context()` loads the top-3 for the routed primary topic. Generated by `corpus/synthesis_scripts/` from Stage-1 extractions; do not edit by hand — re-run the pipeline. |
